# Governance Index

This directory contains the editorial, disclosure, citation, and review
controls that govern public-facing repository work.

## Read in this order

1. [evidence-boundaries.md](evidence-boundaries.md)
2. [public-disclosure-boundary.md](public-disclosure-boundary.md)
3. [citation-policy.md](citation-policy.md)
4. [publication-review-checklist.md](publication-review-checklist.md)

Read the remaining files only when the task needs them:

- [ip-boundaries.md](ip-boundaries.md) for IP or disclosure-boundary questions
- [maturity-taxonomy.md](maturity-taxonomy.md) for internal maturity language
- [governance-mechanics-v0.1.md](governance-mechanics-v0.1.md) for controlled
  conceptual governance mechanics

## Purpose by file

- `evidence-boundaries.md` — the primary gate for claims, maturity language,
  and what may enter public-facing text
- `public-disclosure-boundary.md` — what the public repo may describe about
  Axodus institutionally
- `citation-policy.md` — how sources move from candidate to verified use
- `publication-review-checklist.md` — minimum publication-readiness checks
- `ip-boundaries.md` — intellectual-property and disclosure controls
- `maturity-taxonomy.md` — internal taxonomy guardrails
- `governance-mechanics-v0.1.md` — controlled public conceptual governance
  stub, not an operating manual

## Agent use rules

- Treat this directory as control-plane documentation, not manuscript content.
- Read `evidence-boundaries.md` before drafting or editing public text.
- Do not infer operational maturity, deployment, validation, or authority
  from these files.
- Do not treat `governance-mechanics-v0.1.md` as proof of deployed governance.

## Current status

These files are active controls. None of them authorize publication,
submission, controlled external review, or external circulation by
themselves.
