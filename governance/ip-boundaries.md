# Intellectual-Property Boundaries

Status: Phase 1 editorial control

## Allowed

- high-level conceptual components and relationships;
- research questions and evaluation proposals;
- public literature synthesis;
- non-sensitive governance principles;
- explicit limitations and prototype-stage disclosures;
- conceptual diagrams that cannot be used to reconstruct an implementation.

## Prohibited

- source code or pseudocode that reveals proprietary mechanisms;
- credentials, keys, tokens, secrets, endpoints, or environment data;
- private repository names, paths, links, or access arrangements;
- deployment topology, infrastructure configuration, security controls, or
  exploitable operational detail;
- non-public commercial terms, treasury data, token mechanics, partner data,
  or personal data;
- implementation-specific agent prompts, orchestration rules, or risk
  thresholds not approved for disclosure.

## Review rule

When uncertain, replace implementation detail with an interface-level or
research-level description and record the omission in the readiness report.
Absence of a confidentiality label does not make information public.
