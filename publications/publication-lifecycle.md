# Publication Lifecycle

The lifecycle is reusable across publication series. A document may advance
only when the evidence and authorization for the current state are recorded.

```text
Draft -> Internal Review -> Editorial Review -> Approved -> Released -> Archived
```

## States

- `draft`: source is being prepared; release is not authorized.
- `internal_review`: authors or maintainers are checking scope, evidence, and
  source fidelity.
- `editorial_review`: publication metadata, claims, references, disclosure,
  presentation, and venue requirements are being checked.
- `approved`: the exact version is approved for release but has not necessarily
  been deployed to a public channel.
- `released`: the exact version has an authorized public publication record.
- `archived`: the version is preserved for traceability and is no longer the
  current release.

Lifecycle state and release authorization are separate fields. A draft can be
technically complete while remaining unauthorized for publication.
