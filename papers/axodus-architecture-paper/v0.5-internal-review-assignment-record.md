## Confidentiality Rule

This record must remain confidential internal material.

Do not record public-facing reviewer names, email addresses, handles, or any
other broadly shareable identity marker here unless a later private
assignment-update phase explicitly requires it and the distribution context is
already controlled.

Until then, use:

- reviewer roles;
- assignment status values; and
- optional private reviewer identifiers only if needed for package logistics.

## Allowed Status Values

Use only the following assignment-status values:

- `unassigned`
- `assigned`
- `returned`

If a later phase needs richer tracking, extend the schema only with explicit
authorization.

## Package-Control Checklist for Assignment

Before any row is changed from `unassigned` to `assigned`, confirm:

- [ ] the package version-lock hashes are current;
- [ ] the author pre-distribution checklist is complete;
- [ ] the reviewer is internal and approved for confidential access;
- [ ] the correct role-specific form is attached;
- [ ] the package channel is private and controlled;
- [ ] the reviewer understands that the package is internal-only;
- [ ] no external-review, publication, submission, or circulation authority is
      implied.

## Recording Rules

When a package is assigned:

- change the role status from `unassigned` to `assigned`;
- record the package sent date;
- record a private reviewer identifier only if needed for tracking;
- record the expected return due date; and
- note any role overlap if one reviewer covers more than one role.

When a review form is returned:

- change the role status from `assigned` to `returned`;
- record the return received date; and
- do not summarize findings here. Findings belong in the returned review form
  and a later consolidation record.

## Role Overlap Rule

One reviewer may cover more than one role only if:

- the reviewer’s expertise is adequate for each assigned role; and
- the package owner records that overlap in the `Notes` column.

Role overlap should be minimized when it would reduce independence of review
across architecture, evidence, and evaluation.

## Finding and Decision Separation

This record is only for assignment tracking.

Do not use it to record:

- findings;
- severity judgments;
- author decisions;
- manuscript patch decisions; or
- readiness advancement decisions.

Those belong in:

- the returned role-specific review forms; and
- a later consolidation record.

## Minimum Completion Condition

The assignment cycle is not complete until:

- each required role is either `returned` or explicitly marked as not used in
  a later authorized update;
- all distributed copies are accounted for; and
- the package owner is ready to open a separate consolidation phase.

## Forbidden Actions in This Phase

This assignment-record phase must not:

- fill review findings;
- name reviewers publicly;
- authorize controlled external review;
- authorize publication or submission;
- authorize external circulation; or
- modify the manuscript or bibliography.
