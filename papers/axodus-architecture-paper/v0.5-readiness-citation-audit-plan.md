## Purpose

This document plans a controlled readiness and citation audit for the current
`v0.5` manuscript state. Its purpose is to:

- audit the current readiness status of the `v0.5` line;
- verify the citations effectively used in `paper.md` and `paper.tex`;
- identify whether any orphaned bibliography entries exist in
  `references.bib`;

  manuscript or bibliography in this phase.

This phase does not modify `paper.md`, `paper.tex`, or `references.bib`.

## Audit Scope

The later readiness-and-citation audit opened from this plan should check:

- whether the current `v0.5` readiness framing is internally consistent;
- whether any readiness labels still reflect older manuscript states in ways
  that deserve cleanup;
- which scholarly citation keys are effectively used in `paper.md` and
  `paper.tex`;
- whether `references.bib` contains any entries not cited by either
  manuscript format;
- whether citation usage remains synchronized across Markdown and LaTeX; and
- whether residual sensitive-claim phrasing remains only as boundary-preserving
  negatives or still needs targeted cleanup.

## Current Readiness Findings

Current readiness status remains intentionally blocked:

- `v0.5` is an active internal drafting track only;
- peer review is not authorized;
- publication is not authorized; and
- external circulation is not authorized.

Current readiness records remain broadly consistent with those blockers.

The main readiness issue identified for later review is documentary lag rather
than authorization drift. Some readiness headers still refer to historical
baseline states such as `draft v0.4` because the repository preserves the
v0.4 line as the historical manuscript baseline while the active writing line
is `v0.5`. This is not currently a publication authorization problem, but it
should be reviewed in a later readiness-cleanup phase for clearer status
presentation.

### Markdown extraction caveat

Raw `@...` extraction from Markdown also captures contact-style strings such
as email handles. In the current manuscript this produces false positives
such as:

- `gmail`
- `mzfshark`
- `thinkincoin`

These are not citation keys and must not be treated as bibliography problems.
Any later citation audit must distinguish scholarly `@citationkey` usage from
metadata or contact strings in the manuscript front matter.

### Orphan bibliography status

No orphaned bibliography entries were identified at this stage. The current
`references.bib` entries are all used by at least one manuscript format.

## Current Sensitive-Claim Findings

Residual sensitive-claim terms still appear in the manuscript, but the
current scan indicates that they are mostly being used in one of two bounded
ways:

- as explicit negative or limiting statements; or
- as internal maturity or evaluation terminology that still deserves later
  review for clarity.

Examples that should be reviewed later, but not edited in this phase, include
uses of:

- `deployed`
- `validated`
- `operational`
- `adoption`
- `production operation`

The later audit should distinguish between:

- acceptable boundary-preserving negatives such as “does not imply a deployed
  federation”; and
- wording that may still sound too close to platform-readiness or operational
  framing even when not intended as a claim.
